Setup Document Management¶

Note

In order for Document Management features to appear in the UI, the OmniaManagementSystem extension needs to be installed and the Controlled Documents feature needs to be activated. Please consult with your local partner to set up these prerequisites.

Below you will find the steps needed in order to create a basic setup of Document Management.

Document Types¶

  1. Go to the SharePoint admin center.

  2. Click on Content Services > Term store in the left-hand navigation.

  3. Make sure to add the Working Languages that you would like to support in the Document Management System. Note! This is a global setting. It will affect all services dependent on the term store.

  1. Create a new term set for Document Types.

  1. Go to Omnia Admin > Properties > Enterprise Properties.

  2. Find the property Document Type and click on the edit icon.

  3. Connect the property to the terms set Document Types and save.

Note

This activity can’t be undone in the UI. If the property needs to be reconnected, a support ticket needs to be submitted.

Document Title¶

  1. Go to SharePoint Admin Center.

  2. Click on More features > Search.

  3. Click on Manage Search Schema.

  4. Filter on “refinablestring50”.

  1. Edit the managed property and map the following crawled properties.

  1. Go to Omnia Admin > Properties > Enterprise Properties.

  2. Find the property Title and click on the edit icon.

  3. Check the SharePoint Searchable checkbox and use RefinableString50 as the default Managed Property and check all search boxes.

Document Archive¶

Note

If you have Process Management setup, you can ignore step 16 - 17.

  1. Go to the Business Profile in which you want to host the central controlled documents archive.

  2. Go to Teamwork > Templates and create a new template that can be used for system sites.

  1. Click on Teamwork > Apps and create a new site for the controlled documents archive based on the System Site template.

  1. Activate the features Archived Documents and SPFx Infrastructure on the newly created site.

  2. Click on Document Management in Omnia Admin and click on Settings.

  3. Put in the full url to the newly created site in the Archive Site Url and save.

Document Authors¶

  1. Go to Omnia Admin > Document Management > Settings.

  2. Take a decision on how you want to define controlled documents authors within your solution:

  • Document Authors Group: This will create an explicit SharePoint Group on the site that is used as the document authors group.

  • Site Owners: You need to have the role of the site owner (or M365 Group owner) to be able to edit controlled documents.

  • Site Owners and Members: You need to be at least a site member (or M365 Group member) to be able to edit controlled documents.

  1. Check the Approvers Group checkbox if you want to allow sites to have a specific list that can be used to define a group of site specific approvers.

Basic Document Template¶

  1. Go to Omnia Admin > Document Management > Document Templates.

  2. Create a new Document Template for a blank Word document.

Basic Property Set¶

  1. Go to Omnia Admin > Properties > Enterprise Properties.

  2. Create a new person property for Document Owner.

  1. Go to Omnia Admin > Properties > Property Sets.

  2. Create a new property set for a basic controlled document.

  3. Add the Document Owner property and make it required.

Basic Document Type¶

  1. Go to Omnia Admin > Document Management > Document Types.

  2. Create a first document type that can be used for templates in the system.

First Team Site¶

Note

Make sure you have installed Omnia SPFx Integration using the instruction in Omnia Admin > System > SPFx Instruction.

In order to create a first controlled document that can be indexed by SharePoint Search, we need to create a first site. If you already have existing team sites in your solution that are attached to Omnia, you can use one of those and activate the feature Controlled Documents Library on that site. In this example we will create a new provisioning template for Departments that will activate this feature by default.

  1. Go to Omnia Admin and select your default business profile.

  2. Click on Teamwork > Templates.

  3. Create a new provisioning template for departments.

  1. Create a new site based on the provisioning template.

  1. Go to the newly created site and click on Controlled Documents in the left-hand navigation

  2. Click on Permissions in the right-hand corner of the controlled documents library.

  1. Add a default readers group (in this example Everyone except External Users).

Publish a Controlled Document¶

  1. Create and publish a new controlled document.

  1. Create a draft and publish a new edition. (Note! We do this to be able to crawl a document in the archive as well).

Default Search Properties¶

Now you will have to wait a few minutes before the controlled document has been indexed by SharePoint Search.

  1. Go to SharePoint admin center > More features > Search.

  2. Click on Managed Search Schema.

  3. Map the following Managed and Crawled Properties. (Note! This is just a recommendation and any refinable managed properties can be used.)

  • RefinableString51 > ows_ODMDocId

  • RefinableString52 > ows_ODMDocumentType

  • RefinableString54 > ows_ODMApprovedBy

  • RefinableString55 > ows_ODMContentLanguage

  • RefinableString56 > ows_SiteName

  • RefinableString57 > ows_SPSiteUrl

  • RefinableString59 > ows_q_USER_ODMApprovedBy

  • RefinableString60 > ows_taxid_ODMDocumentType

  • RefinableInt30 > ows_ODMDocIdNumber

  • RefinableInt31 > ows_ODMEdition

  • RefinableDate10 > ows_q_DATE_ODMPublished

  • RefinableDate11 > ows_q_DATE_ODMApproved

  • RefinableDate12 > ows_q_DATE_ODMReviewDate

  • RefinableDate13 > ows_q_DATE_ODMPublishedFrom

  • RefinableDate14 > ows_q_DATE_ODMPublishedTo

  1. Go to Omnia Admin > Properties > Enterprise Properties.

  2. Edit and update the search configuration for the following enterprise properties.

  1. Create another draft of a controlled document and publish it. (Note! This is necessary in order for SharePoint Search to map the crawled properties to the managed properties.)

You are now ready to start creating Document Types, Document Templates and add Controlled Documents in Teamwork and Publishing Apps.
